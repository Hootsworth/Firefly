01

Product clarity

Firefly should never feel like a magic benchmark. It is a transfer diagnostician, a hardware fact collector, and a safe benchmark runner.

Transfer diagnosis

Explain why a process took longer than its modeled path suggests.

Output: expected t, observed t', delta, bottleneck stack, and next checks.
Hardware ground truth

Collect host facts before asking the user to fill in assumptions.

Output: measured OS facts, redacted identifiers, unavailable counters, confidence level.
Benchmark engine

Run bounded tests that isolate memory, burst I/O, and queue pressure.

Output: elapsed time, service rate, safety note, and model mapping.
02

Truth labels

Every number needs provenance. Firefly separates measured facts from derived rates, user assumptions, and privileged data it cannot access.

Measured

Read from the host or produced by a benchmark in this session.

CPU model, storage protocol, RAM copy throughput.
Derived

Calculated from measured facts and deterministic protocol rules.

PCIe ceilings after encoding overhead.
Assumed

Entered by the user or defaulted because the OS did not expose it.

SLC cache size, competing PCH traffic.
Blocked

Requires elevated APIs, vendor tooling, or signed native drivers.

Some thermals, memory-controller counters, direct bus queues.
